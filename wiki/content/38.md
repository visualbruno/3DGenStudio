# Mesh

Export a single mesh to a standard 3D file format, with engine-ready presets for Blender, Unity and Unreal. This is how you take a finished asset out of 3DGenStudio and into a game engine or another DCC tool.

## Where to export from

- **Mesh preview** — open a mesh asset's preview, then click **Export**. This path can carry the asset's **rig and animation clips**, so it offers the full set of engine presets.
- **Mesh Editor** — the editor can also export the mesh you're working on. Because the editor hands over rig-free geometry, only the plain geometry formats (GLB, OBJ, PLY, STL) are offered there.

## Exporting

1. Click **Export** to open the export dialog.
2. Choose a **format** (see below).
3. Set the **file name** — the extension is filled in automatically for the chosen format.
4. Choose an **output folder** — type the path or click **Browse**.
5. Click **Export**. A progress bar is shown for formats that need conversion, and a success message lists the files that were written.

## Formats

| Format | File | Notes |
| --- | --- | --- |
| **GLB** | `.glb` | Single file with textures embedded. The default. |
| **Blender — GLB (rig + animations)** | `.glb` | Blender imports GLB natively (*File → Import → glTF 2.0*) with the skeleton and every animation clip. GLB source assets are copied byte-for-byte for perfect fidelity. |
| **Unity — FBX (rig + takes)** | `.fbx` | Drop the `.fbx` into *Assets*. Textures are embedded — use *Materials → Extract Textures*. Pick the rig type under *Rig* (Humanoid may need *Enforce T-Pose*); each clip appears as a separate take. |
| **Unreal Engine — FBX (cm, rig + takes)** | `.fbx` | Import as a *Skeletal Mesh* with *Import Animations* enabled. Exported in centimeters at scale 1 — no unit conversion needed. |
| **FBX — generic (rig + takes)** | `.fbx` | Neutral FBX (meters, Y-up) with the skeleton and one take per clip. Suitable for Godot, Maya, 3ds Max and other DCC tools. |
| **OBJ — geometry + .mtl + textures** | `.obj` | Saves geometry, materials and textures as separate files named after the mesh (e.g. `mesh.obj`, `mesh.mtl`, `mesh_albedo.png`). |
| **PLY** | `.ply` | Geometry only. |
| **STL** | `.stl` | Geometry only. |

> The **FBX presets** (Unity, Unreal, generic) convert through the **Mesh Tools service**, which the app starts automatically when you export. GLB and the geometry-only formats (GLB, OBJ, PLY, STL) export directly and need no service.

## Rig and animations

The FBX and Blender presets exist specifically to carry a mesh's **skeleton and animation clips** into an engine. When the source asset is already a `.glb`, its original bytes are used so skin weights, clips and textures are preserved exactly. Otherwise the mesh is serialized through three.js, which still carries the loaded animations onto the export. After an FBX conversion the success message reports the bone and clip counts that were written.
