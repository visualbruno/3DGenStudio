# Project

Export and import entire projects as self-contained **`.3dgp` bundles**. A bundle carries the project *and* every asset it links to — meshes, images, sub-assets and thumbnails — so you can back it up, move it to another machine, or hand it to a teammate.

Both actions are available from the **Projects** page.

## Exporting a project

1. On the **Projects** page, hover a project card and click its **Export** action.
2. Give the export a **name** (defaults to the project's name).
3. Choose a **destination folder** — type the path or click **Browse**.
4. Click **Export**.

The app creates a folder named after the export containing:

```
<name>/
  <name>.3dgp     ← the project bundle file
  assets/         ← every linked asset, sub-asset and thumbnail
```

On success the dialog reports how many assets and files were written, and where.

## Importing a project

1. On the **Projects** page, click **Import**.
2. Choose the **source folder** — the folder that contains the `.3dgp` file. Type the path or click **Browse**. (When you browse to a folder, its name is used as the default project name.)
3. Optionally set a **project name**. Leave it blank to keep the name stored in the bundle.
4. Click **Import**.

Every asset in the bundle is copied in as a **fresh library entry** and a **brand-new project** is created from it — the import never overwrites or merges with an existing project. When it finishes, the app opens the newly imported project.

---

To export a single mesh (rather than a whole project) to a 3D file format, see **[Mesh export](38.md)**.
