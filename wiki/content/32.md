# Auto Rig

Generate a full skeleton and skin weights for your mesh automatically. Auto Rig sends the current mesh to the **SkinTokens / TokenRig** rigging service, which predicts a humanoid skeleton, binds it to the surface with skin weights, and returns a **rigged (skinned) GLB**. The skeleton is then shown as an overlay in the viewport.

> Auto Rig runs on the **SkinTokens rigging service**. Configure and start it in **Settings → Rigging** before using this mode. It needs an **NVIDIA GPU**.

Unlike Auto UV and Auto Retopo, the rig result is **not** flattened back into the editable mesh — its value *is* the skeleton. So instead of Keep/Revert, you **Save as version** or **Download** the rigged GLB.

## Running it

1. Open the **Auto Rig** mode in the left toolbar.
2. Tune the **Skeleton** and (optionally) **Generation** options below.
3. Click **Run Auto Rig** and wait for the service to finish. A progress bar reports its state.
4. When it completes, the skeleton appears on the mesh and a **Rig generated** panel shows the result metrics (bone count, etc.).
5. Choose what to do with the result:
   - **Save as version** — store the rigged GLB as a new version of the asset in your library.
   - **Download** — save the rigged GLB to disk.
   - **Dismiss** — hide the result panel (the skeleton overlay stays visible).

## Show skeleton

The **Show skeleton** toggle overlays the bones on the mesh in the viewport. It is enabled whenever the current mesh has a skeleton — either one you just generated, or one that was already present on a loaded rigged mesh. If the mesh has no skeleton yet, the toggle is disabled and a hint prompts you to run Auto Rig.

## Skeleton options

Control the skeleton that gets generated.

| Field | Default | What it does |
| --- | --- | --- |
| **Bone names** | Mixamo | Rename the generated bones to a standard humanoid convention for retargeting. Choose **Mixamo**, **Unreal Engine 5**, or **Keep model names**. |
| **Preserve texture & scale** | On | Transfer the rig onto your *original* mesh so it keeps its texture and scale. Recommended — leave on. |
| **Voxel-skin postprocess** | On | Clean up skin weights with a voxel pass to reduce weight bleed across disconnected parts. |
| **Keep model loaded in memory** | On | Keep the rig model in (GPU) memory for fast repeat rigs. Turn off to free memory after each rig — the next rig reloads it (slower first run). |

## Generation (advanced)

These control the sampling of the skeleton-prediction model. The defaults are tuned for good results — only adjust them if you need to experiment with the output.

| Field | Range | What it does |
| --- | --- | --- |
| **Top-k** | 1–200 | Top-k sampling limit. |
| **Top-p** | 0.1–1 | Nucleus (top-p) sampling threshold. |
| **Temperature** | 0.1–2 | Sampling randomness. Lower is more deterministic. |
| **Repetition penalty** | 0.5–3 | Penalizes repeated tokens during generation. |
| **Beams** | 1–20 | Beam-search width. |

---

The result is a **skinned mesh** — save it as a new version to keep the rig. Once a mesh is rigged, you can open the **[Skeleton](33.md)** panel to inspect its bones and the **[Animations](34.md)** tab to retarget motion onto it.
