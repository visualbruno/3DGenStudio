# Bone Mapping

Before a reference animation can be retargeted onto your mesh, the app needs to know which bone on your mesh corresponds to each bone in the reference (animation) skeleton. The **Bone Mapping** dialog is where you set that up. It opens from the **[Animations](34.md)** tab via **Map bones** (or **Edit mapping** if a mapping already exists).

Each row maps one **source** bone (the reference animation skeleton, left) to one **target** bone (your rigged mesh, right). A source bone can be used at most once.

## Layout

The dialog has three parts:

- **3D skeleton views** (far left) — two live, rotatable skeletons: the **source** (reference) on top and your **target** mesh below. These make it obvious *where* each bone sits, which names alone rarely tell you.
- **Source column** — every bone in the reference animation skeleton, with a filter box.
- **Target column** — every bone in your mesh, with a filter box and the mapping actions.

The header and footer both show how many bones are currently mapped (e.g. *18 of 22 target bones mapped*).

## Mapping bones

There are several ways to create a mapping:

- **Drag** a source bone onto a target row.
- **Click a source** bone, then **click a target** — the source is assigned to that target.
- **Click a bone in either 3D view** to select it (same as clicking its row). If a source is already picked, clicking a target bone in the 3D view assigns it.
- **Hover a list row** to flash that bone in the corresponding 3D view.

Mapped source bones are marked with a check and greyed as "used". Each target row shows the source bone it is bound to as a chip, or **unmapped** if none. Click the **×** on a mapped target row to clear it.

## Actions

| Button | What it does |
| --- | --- |
| **Auto-Map** | Fill the mapping heuristically from scratch, **replacing** the current mapping entirely. |
| **Finish Mapping** | Fill only the *currently unmapped* target bones from the heuristic, **keeping** the ones you've already set and never reusing a source that's already in use. Disabled once every target is mapped. |
| **Clear** | Remove all mappings. |

## Saving

Click **Save mapping** to apply it. The button is enabled once at least one bone is mapped. After saving, the reference's animation clips load into the Animations tab, ready to preview and retarget. Press **Escape**, click **Cancel**, or click outside the dialog to close without saving.
