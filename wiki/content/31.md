# Optimize

Reduce the triangle count of your mesh while preserving its shape. Optimize runs the bundled **gltfpack** (meshoptimizer) binary to simplify the geometry — perfect for producing lighter versions for real-time use or export.

> Unlike Auto UV and Auto Retopo, Optimize does **not** require the Python mesh-tools service. It uses the gltfpack binary shipped with the app.

## Running it

1. Open the **Optimize** mode in the left toolbar.
2. Set the **Simplify ratio** to the fraction of triangles you want to keep.
3. Click **Run Optimize** and wait for it to finish.
4. The result replaces the current mesh and reports the applied ratio. Click **Keep** to accept it, or **Revert** to restore the previous mesh.

Optimize preserves the existing UV layout — it only removes geometry, so any texturing you've already done stays intact.

## Parameters

### Simplify

| Field | Default | What it does |
| --- | --- | --- |
| **Simplify ratio** | 0.5 | Target triangle count as a fraction of the original (1 = no simplification, 0.5 = roughly half the faces). |

When the current mesh has a known face count, the panel shows a live estimate:

- **Current faces** — the mesh's face count before optimizing.
- **Target faces** — the approximate count after optimizing (`current faces × simplify ratio`).

For example, with a 100,000-face mesh and a ratio of `0.1`, the target is roughly ~10,000 faces.

---

The result replaces the mesh; use **Keep** or **Revert** to decide.
